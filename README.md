# spnl-erb

[matz/spinel](https://github.com/matz/spinel) 向け **ビルド時 ERB → Ruby コンパイラ**.
Crystal の [`ECR`](https://crystal-lang.org/api/master/ECR.html) と同じ思想で,
テンプレートを **コンパイル前に static Ruby に変換**する.

## なぜ必要か

Spinel は AOT コンパイラで `eval` を持たないため, CRuby 標準の ERB は実質使えない:

```ruby
# Spinel 標準の lib/erb.rb は placeholder (matz 自身が明記)
class ERB
  def result_with_hash(vars)
    @input    # ← テンプレートそのまま返すだけ
  end
end
```

ERB はそもそも **「Ruby を生成する文法糖」** なので, 生成側 (build time) と
実行側 (runtime) を分離すれば AOT に乗せられる:

```
foo.html.erb                spnl-erb compile           foo.html.generated.rb            spinel compile
"<h1><%= @x %></h1>"  ───────────────────────────►   _out = _out + "<h1>" + @x  ─────────────────────►  native binary
        (CRuby で実行)                                          (Spinel で実行)
```

## インストール

依存: CRuby (任意のバージョン, ビルド機にのみ必要).

```sh
git clone https://github.com/gorin9/spnl-erb.git ~/.local/spnl-erb
sudo ln -s ~/.local/spnl-erb/bin/spnl-erb /usr/local/bin/spnl-erb
```

または `spinel-packer install gorin9/spnl-erb` で vendor 取得.

## 使い方

### 単一ファイル

```sh
$ spnl-erb views/index.html.erb -c IndexView -a name,items
# AUTO-GENERATED by spnl-erb — DO NOT EDIT
class IndexView
  def initialize(name, items)
    @name = name
    @items = items
  end
  def render
    _out = ""
    _out = _out + "<h1>"
    _out = _out + (@name).to_s
    ...
    _out
  end
end
```

### ディレクトリ一括 (推奨)

```sh
$ spnl-erb --batch views/
wrote views/users/index.html.generated.rb (attrs: list)
wrote views/users/show.html.generated.rb (attrs: user)
wrote views/posts/show.html.generated.rb (attrs: post, comments)
```

クラス名・ivar 名は **path とテンプレート内容から自動推論**:
- `views/users/index.html.erb` → `class UsersIndexView`
- テンプレ中の `@list` `@user` 等を scan して initialize の引数に

### 利用側

```ruby
# app.rb
require_relative "views/users/index.html.generated"

v = UsersIndexView.new(["alice", "bob"])
puts v.render
```

Spinel でコンパイル:
```sh
$ spinel app.rb -o bin/app
$ ./bin/app
<h1>Users</h1>
<p>alice</p>
<p>bob</p>
```

## 大きめのサンプル: `examples/blog/`

ブログ風サイトの **multi-view + layout + model** 統合例 (400+ 行).
4 つのテンプレート, 2 モデル, 1 ランナー (app.rb) からなり, AOT compile 後の
単一バイナリが 3 ページの HTML を 16 ms で書き出す.

```sh
cd examples/blog/
RUBY=ruby ./build.sh
spinel app.rb -o blog_app
./blog_app
open out/index.html
```

詳細は [`examples/blog/README.md`](examples/blog/README.md) 参照.

### 重要: stateless mode をデフォルトに

Spinel は **`@var` に `Array<UserClass>` を保持すると SIGSEGV** する
(blog example で発覚した制約).
このため spnl-erb は `--mode stateless` をデフォルトとし, `class + ivar` ではなく
**`module + class method`** を出力する:

```ruby
# ✅ デフォルト (stateless)
module PostsIndexView
  def self.render(posts, total_count)
    _out = ""
    _out = _out + posts.length.to_s
    posts.each { |p| ... }     # 局所変数, @ 無し
    _out
  end
end

# 利用側
PostsIndexView.render(posts, 42)
```

テンプレ作者は `@posts` と書いても OK — generator が局所変数に書き換える.

旧 `class + initialize + render` 形式が必要なら `--mode instance` で出せる
(ivar が安全な単純型のみ使う場合).

## テンプレ内で使える Ruby

spnl-erb は **タグ内 (`<%= %>` `<% %>`) を pass-through で出力**するため,
**Spinel がコンパイルできる Ruby なら何でも書ける**.

### ✅ 確認済み (動作する)

| 文法 | 例 |
|---|---|
| `if` / `elsif` / `else` / `end` | `<% if @users.length > 0 %>...<% end %>` |
| `case` / `when` (Symbol も可) | `<% case @status; when :active then ... %>` |
| 三項演算子 | `<%= n > 0 ? "yes" : "no" %>` |
| ループ `each` | `<% @items.each do |it| %>...<% end %>` |
| Method chain | `<%= @users.map { |u| u.upcase }.join(", ") %>` |
| 数値計算 | `<%= a / b %>`, `<%= n % 2 == 0 ? "even" : "odd" %>` |
| `String#downcase` / `gsub` / `strip` | `<%= @title.gsub("a", "b") %>` |
| Hash with **String** key + `each` | `<% @meta.each do |k,v| %><%= k %>: <%= v %><% end %>` |
| Symbol case dispatch | `case s; when :foo` |
| Local var across tags | `<% i = 0 %>...<% i = i + 1 %>...<%= i %>` |

### ❌ 避けるべき (Spinel の既知制約)

| 文法 | 症状 | 代替 |
|---|---|---|
| **`Array#select` / `#filter`** | segfault | `each + push if` で書き換え |
| **Symbol を Hash の VALUE に** | 取り出すと `0` | String にする |
| **Symbol key の Hash#each で `|k,v|` の k** | int 化 | `h.keys.each` 経由 |
| `String#sub` の `\1` `\2` backref | 壊れる | 手動 split + 連結 |
| `String#chomp("suffix")` | arg 無視 | `s[0, s.length-1]` |
| `String#scan(/(\w+)/)` の captures | whole match のみ | match + loop |
| `class Array` reopen | dispatch しない | module helper |
| polymorphic Array push | type error | 同じクラスに統一 |

→ 詳細は [`spnl-support` の README](https://github.com/gorin9/spnl-support#踏んだ-spinel-制約)

## ERB 文法サポート

| 文法 | サポート | メモ |
|---|---|---|
| `<%= expr %>` | ✅ | `(expr).to_s` に展開 |
| `<% code %>` | ✅ | コード直接埋め込み |
| `<%# comment %>` | ✅ | 無視 |
| `<%- ... %>` | ✅ | 直前 literal の末尾改行を削る |
| `<% ... -%>` | ✅ | 直後 literal の先頭改行を削る |
| `<%== %>` (raw) | ❌ | 未対応 (`<%= raw(x) %>` で代用) |

## 生成コードの Spinel 互換性

意図的に以下を**使わない**ように出力:

- `+""` (mutable string literal) — Spinel が `char *` で混乱
- `String#<<` — `+=` で代替
- `.freeze` — 不要 (Spinel は frozen-string-literal を意識しない)
- `eval` / `binding` — 当然不可

代わりに:
- `_out = ""` (普通の literal)
- `_out = _out + "..."` (concat)
- ivar (`@x`) と method (`x.to_s`) のみ

## API (CRuby library として使う場合)

```ruby
require_relative "spnl-erb/lib/spnl_erb"

# body だけ
SpnlErb.compile_body("<h1><%= @x %></h1>")
# => "_out = \"\"\n_out = _out + ...\n_out"

# class wrapper (default: stateless module + class method)
SpnlErb.compile_class(template,
  class_name: "UserView",
  attrs: ["name", "email"])

# ivar 形式が要るとき
SpnlErb.compile_class(template,
  class_name: "UserView",
  attrs: ["name", "email"],
  mode: :instance)
```

## ワークフロー統合

### Makefile

```make
ERB_FILES := $(shell find views -name '*.erb')
GENERATED := $(ERB_FILES:.erb=.generated.rb)

%.generated.rb: %.erb
	spnl-erb --batch $(dir $<)

build: $(GENERATED)
	spinel app.rb -o bin/app

clean:
	find views -name '*.generated.rb' -delete
```

### `.gitignore` 議論

`*.generated.rb` をコミットするか?
- **コミット派 (推奨)**: schema / template の変更が PR 差分で見える
- ignore 派: build 時に再生成すれば同じ

Rails の `db/schema.rb` 流の **コミット派**でいきたい.
